# 双向链接

> 如果你在笔记 A 中引用了笔记 B，笔记 B 页面上也会自动显示「A 引用了你」— 这就是双向链接

## 语法

在 [[Obsidian]] 中，用 `[[笔记名称]]` 创建链接。笔记可以还不存在 — 你先链接，以后再写。

```markdown
参见 [[卡片笔记法 Zettelkasten]]
请参考 [[Python 学习路线|Python]] 的第 3 章
```

`|` 后面是显示别名：地址是 `Python 学习路线`，但页面上只显示 `Python`。

## 传统链接 vs 双向链接

| | 传统超链接 | 双向链接 |
|---|-----------|---------|
| 方向 | 单向：A → B | 双向：A ↔ B |
| B 的感知 | B 不知道自己被引用 | B 的反向链接面板显示 A |
| 断链感知 | 断裂后无感知 | 可发现孤立笔记 |
| 导航 | 只能前进 | 前进 + 退回 |

## 三种用法

### 1. 关联 — 说"这个跟那个有关"

```markdown
<!-- 在 Obsidian 笔记里 -->
长期使用 [[卡片笔记法 Zettelkasten]] 会发现，
分类会自然浮现，就像 [[知识图谱]] 中的聚类。
```

### 2. 对比 — 说"这个跟那个不同"

```markdown
和 [[C 语言]] 的手动内存管理不同，
[[Python 基础语法|Python]] 自动处理垃圾回收。
```

### 3. 来源 — 说"这个想法来自那里"

```markdown
这个技巧来自 [[第二大脑]] 的 CODE 方法论。
```

## 反向链接面板

打开任意笔记，看右侧（或 `Ctrl+P` → "反向链接"），你会看到：
- **链接到当前页面** — 哪些笔记提到了它
- **未链接提及** — 文本里出现了笔记名但没加 `[[]]`

> 双向链接是 [[卡片笔记法 Zettelkasten]] 得以运行的基础设施。没有它，你永远不知道谁引用了谁。
