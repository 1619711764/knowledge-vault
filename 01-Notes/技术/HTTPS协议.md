# HTTPS 协议

> HTTP + TLS/SSL 加密 = HTTPS。HTTP 是明信片（谁都能看），HTTPS 是保险箱（只有收发双方能打开）

## HTTP vs HTTPS

| | HTTP | HTTPS |
|---|------|-------|
| 加密 | 无，明文 | TLS/SSL 加密 |
| 端口 | 80 | 443 |
| 标识 | 不安全警告 | 小锁图标 |
| 速度 | 略快 | 略慢（加解密开销） |

## TLS 握手（简化）

```
客户端                          服务器
  │── ClientHello ─────────────→│  "我支持这些加密方式"
  │←─ ServerHello + 证书 ───────│  "用这个，这是我的身份证"
  │── 验证证书 + 生成会话密钥 ──→│
  │←──── 加密通信 ──────────────→│  之后所有数据加密传输
```

## 证书

服务器发来的"身份证"，由 CA（证书颁发机构）签发。浏览器内置受信任 CA 列表，验证通过才显示小锁。

## 面试话术

> 问："HTTP 和 HTTPS 的区别？"
> 答："HTTPS = HTTP + TLS/SSL 加密层。保证传输不被窃听、篡改、冒充。端口 443，需要证书。浏览器标记 HTTP 为不安全。"
