# 内网服务器开发

> "内网服务器开发经验" — 面试中常见的考察点

## 什么是内网服务器

和公网服务器（任何人都能访问）不同，内网服务器只在公司/组织内部网络可访问。比如：
- 公司内部的 GitLab、Jenkins
- 数据库服务器（MySQL、Redis）
- 内部 API 网关
- 日志收集服务器（ELK Stack）

## 常见的考察点

### 1. 网络基础

面试官问"内网开发"时，实际在问：

| 考点 | 要懂到什么程度 |
|------|-------------|
| TCP/IP | 三次握手、四次挥手 |
| HTTP/HTTPS | 状态码、方法、Header |
| 端口 | 常见端口（22 SSH、80 HTTP、443 HTTPS、3306 MySQL、6379 Redis） |
| DNS | 域名解析过程 |
| 代理/反向代理 | Nginx 的作用 |

### 2. 服务部署

```bash
# 内网服务器上部署一个 Python API
systemctl start my-api        # systemd 管理服务
journalctl -u my-api -f       # 查看日志
ss -tlnp | grep 8080          # 查看端口是否在监听
```

### 3. 数据库交互

内网服务器最常见的工作：连数据库、写接口

```python
# 连接内网数据库
import mysql.connector
conn = mysql.connector.connect(
    host="10.0.1.100",    # 内网 IP
    user="reader",
    password="xxx",
    database="logs_db"
)
```

### 4. 日志和监控

内网服务器必备：
- 日志收集：ELK（Elasticsearch + Logstash + Kibana）
- 监控告警：Prometheus + Grafana
- 日志分析：[[AI 日志解析]] + [[正则表达式实战]]

### 5. 安全常识

内网≠安全。常见安全措施：
- 防火墙规则（只开放必要端口）
- SSH 密钥登录（禁用密码登录）
- 数据库权限最小化（读写分离、限 IP 访问）
- 敏感信息不入代码（用环境变量）

## 面试话术

> 问："你有内网服务器开发经验吗？"
> 答："有。在项目里负责过内网 API 的开发和部署，用 FastAPI 写接口，systemd 管理服务，Nginx 做反向代理。数据库在内网，用过 MySQL 和 Redis。日志用 ELK 收集，出了问题能从日志快速定位。日常操作遵循最小权限原则。"
