# Markdown

> 轻量级标记语言，用纯文本写出有格式的文档。所有 Obsidian 笔记的本质就是 `.md` 文件

## 为什么不用 Word？

| | Word | Markdown |
|---|------|----------|
| 文件格式 | `.docx`（二进制） | `.md`（纯文本） |
| 工具依赖 | 必须用 Office 打开 | 任何文本编辑器 |
| 版本控制 | Git 无法 diff | Git 可以逐行对比 |
| 搜索 | 需索引 | `grep` 直接搜文本 |
| 迁移成本 | 高（格式可能丢失） | 无（纯文本永不过时） |

## 常用语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**加粗** 和 *斜体*

- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2

[链接文字](https://example.com)
![图片描述](image.png)

> 引用块

`行内代码`

​```
代码块
​```

| 表头 1 | 表头 2 |
|--------|--------|
| 单元格 | 单元格 |
```

## Obsidian 专属语法

标准 Markdown 没有以下功能，[[Obsidian]] 自己加的：

```markdown
[[笔记名称]]                  # 双向链接
[[笔记名称|别名]]              # 链接别名
![[笔记名称]]                  # 嵌入另一篇笔记全文

---
key: value                    # YAML Frontmatter（元数据）
tags: [标签1, 标签2]
---

#标题                         # 无序标题
##子标题

- [ ] 未完成任务                # 任务复选框
- [x] 已完成任务
```

## 实用技巧

- **Ctrl+Click** — 在 Obsidian 中打开链接
- 用 `---` 包裹的 YAML 头部叫 **Frontmatter**，[[Dataview]] 就是靠它做查询
- `.md` 文件可以直接拖到 VS Code 编辑，批量修改时比 Obsidian 快

> 你的所有知识库都是 `.md` 纯文本文件。即使 10 年后 Obsidian 不再存在，知识也不会丢失 — 这是选择 [[个人知识管理|Markdown 笔记]] 最重要的原因。
