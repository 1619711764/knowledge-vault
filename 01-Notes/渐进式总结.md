# 渐进式总结

> 不要试图一次性把笔记写完美。像给植物浇水一样，每次回顾笔记时加深一层。

## 五层金字塔

```
         ┌──────┐
         │ 重混  │  ← 第 5 层：跨笔记综合，产出新观点
        ┌┴──────┴┐
        │ 摘要    │  ← 第 4 层：顶部 2-3 句自己的总结
       ┌┴────────┴┐
       │ 高亮      │  ← 第 3 层：标记最核心的句子
      ┌┴──────────┴┐
      │ 加粗        │  ← 第 2 层：加粗关键信息，快速扫描
     ┌┴────────────┴┐
     │ 原始内容       │  ← 第 1 层：摘录原文、粘贴链接
     └───────────────┘
```

## 每层示例

以学习 [[Python 基础语法]] 的笔记为例：

**第 1 层 — 原始摘录：**
> "Python 使用缩进来定义代码块，标准缩进为 4 个空格。与 C 语言不同，Python 没有花括号 {}。"

**第 2 层 — 加粗关键：**
> "Python 使用**缩进**来定义代码块，标准缩进为**4 个空格**。与 C 语言不同，Python **没有花括号**。"

**第 3 层 — 高亮核心：**
> Python 的缩进是语法的一部分，不是可选的。

**第 4 层 — 顶部摘要：**
> Python 用强制缩进替代花括号来定义代码块。这既是新手最大的不同之处，也是 Python 代码可读性高的原因。

**第 5 层 — 重混**
> 把 Python 缩进、[[Markdown]] 语法、YAML 前端元数据三条笔记的精髓合并，写成 [[Python 代码风格|编码规范]]。

## 实践建议

- **别一次做到第 4 层。** 第 1-2 层在一次阅读中完成，过几天回来做第 3 层
- **10 秒测试：** 如果一条笔记 30 秒内扫不到重点，说明需要加粗/高亮
- **重混的触发条件：** 当你发现自己在反复参考某几条笔记时，说明该做第 5 层了

这个方法论来自 [[第二大脑]] 的 CODE 流程中的 "Distill" 阶段：知识不是越存越多就好，越提炼越有价值。
