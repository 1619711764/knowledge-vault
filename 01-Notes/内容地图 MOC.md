# 内容地图 MOC

> Map of Content — 笔记的"目录页"。用 [[双向链接]] 串联同一主题的所有笔记

## 为什么需要 MOC？

你的大脑是**网状结构**，但文件夹是**树形结构**。MOC 是两者的桥梁：

```
文件系统（物理存放）          MOC（逻辑索引）
─────────────────────      ─────────────────
01-Notes/技术/Python/       [[Python 学习路线]]
├── Python 基础语法           ├── Python 基础语法
├── Python 数据类型           ├── Python 数据类型
├── Python 函数与模块         ├── Python 函数与模块
└── Python 常用库             ├── Python 常用库
                              ├── Python 文件操作
                              └── Python 错误处理
```

- 文件系统只关心**位置**
- MOC 只关心**关系**

## 什么时候创建 MOC

当你发现自己在反复查找某个主题的笔记时 — 就是该建 MOC 的时候。一般一个主题积累到 5-10 篇笔记就该整理了。

## MOC 的结构

一个典型的 MOC 页面：

```markdown
# 主题名 — MOC

## 核心概念
- [[概念A]]
- [[概念B]]

## 进阶话题
- [[深入话题1]]
- [[深入话题2]]

## 相关资源
- [[外部参考1]]
```

参考 [[Python 学习路线]] 和 [[Linux 常用命令]] 就是两个 MOC 的实际例子。

## MOC 配合 Dataview

在 MOC 底部加上 [[Dataview]]，自动拉取所有关联笔记：

```dataview
TABLE tags, date
FROM [[]]
SORT date DESC
```

`FROM [[]]` = 查找所有 `[[链接到当前页面]]` 的笔记。MOC 会自动发现新加入的笔记，无需手动更新。

## MOC vs 标签 vs 文件夹

| | 文件夹 | 标签 | MOC |
|---|--------|------|-----|
| 结构 | 树（层级） | 扁平 | 网状 |
| 一篇笔记 | 只能在 1 个文件夹 | 可打多个标签 | 可被多个 MOC 引用 |
| 用途 | 物理存储 | 分类筛选 | 逻辑导航 |

> MOC 是"主动导航"，标签是"被动筛选"。两者互补，不是竞争。
